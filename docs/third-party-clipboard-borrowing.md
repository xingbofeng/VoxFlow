# Clipboard And Pasteboard Borrowing Plan

This document records the open-source code VoxFlow should borrow for the Asset and Palette v1 work. The rule for this feature is: copy proven pasteboard handling where the license allows it, and only rewrite where VoxFlow's domain boundaries require it.

## License Summary

| Project | Repository | License | Use |
| --- | --- | --- | --- |
| Maccy | `https://github.com/p0deje/Maccy` | MIT | Primary source for macOS pasteboard polling, type constants, multi-item extraction, file URL pasteboard writes, and paste keyboard event details. |
| Stash | `https://github.com/hex/Stash` | MIT | Primary source for compact ClipboardMonitor structure, content type priority, source app detection, file/image extraction, and paste service shape. |
| Pasteboard Viewer | `https://github.com/sindresorhus/Pasteboard-Viewer` | Mixed app dependency licenses; source is reference only | Reference for pasteboard type inspection and preview behavior. Do not copy app code into VoxFlow v1 unless separately audited. |

When implementation copies or closely adapts code from Maccy or Stash, keep the original MIT copyright notice in `docs/third-party-licenses.md` and add a short source comment near the copied code.

## Maccy Borrowing

### `Maccy/Clipboard.swift`

Borrowing mode: **partial direct copy plus adaptation**.

Copy or closely adapt:

- `supportedTypes`: `.fileURL`, `.html`, `.png`, `.rtf`, `.string`, `.tiff`.
- `ignoredTypes` constants only for technical markers: `.autoGenerated`, `.concealed`, `.transient`.
- `start()` timer-polling pattern using `NSPasteboard.general.changeCount`.
- `checkForChangesInPasteboard()` structure:
  - return when `changeCount` did not change;
  - read `pasteboardItems`;
  - merge multiple pasteboard items into one logical record;
  - skip empty plain text when no rich text exists;
  - filter dynamic `dyn.` types;
  - filter Microsoft OLE source types;
  - record source app bundle identifier.
- `copy(_ item:)` file URL behavior:
  - write non-file representations with `setData`;
  - write file URLs with `pasteboard.writeObjects`;
  - do not use `writeObjects` for rich text because Maccy documents duplicate paste behavior.
- `paste()` keyboard event details only if VoxFlow's existing `FastPasteTextInserter` is insufficient.
- `isEmptyString(_:)` and `richText(_:)`: direct copy is acceptable.
- `clearFormatting(_:)`: copy if plain-text paste is added; keep file URL preservation.

Do not copy:

- `Defaults`, `Sauce`, `Notifier`, `AppState`, SwiftData `History` integration.
- Maccy pinning logic. Pinning is out of phase-one scope.
- Maccy privacy settings UI. VoxFlow v1 does not special-case copied secrets.

### `Maccy/Extensions/NSPasteboard.PasteboardType+Types.swift`

Borrowing mode: **direct copy of constant raw values, renamed into VoxFlow style**.

Copy constants for:

- `org.nspasteboard.AutoGeneratedType`
- `org.nspasteboard.ConcealedType`
- `org.nspasteboard.TransientType`
- `org.nspasteboard.source`
- `org.p0deje.Maccy` pattern, renamed to VoxFlow's own internal marker such as `com.voxflow.pasteboard.internal`
- Microsoft object/link source types
- Safari/WebKit/Chromium metadata types
- Apple Notes rich text type

VoxFlow must use its own internal marker, not `org.p0deje.Maccy`.

### `Maccy/Models/HistoryItem.swift`

Borrowing mode: **partial adaptation**.

Borrow:

- `transientTypes` idea for excluding unstable metadata from content hashing.
- `supersedes(_:)` concept for deduplication while ignoring transient types.
- `previewableText` order:
  - file URLs;
  - plain text;
  - RTF string;
  - HTML string;
  - generated title.
- image data detection order across TIFF/PNG/JPEG/HEIC if JPEG/HEIC are added.

Do not copy:

- SwiftData model shape.
- OCR-on-image behavior from Maccy. VoxFlow screenshot OCR remains a screenshot-derived capability, not a separate Asset.

### `Maccy/Models/HistoryItemContent.swift`

Borrowing mode: **concept only**.

Use the type/data pair idea for raw pasteboard representations if VoxFlow chooses to persist rich pasteboard data. Do not copy the SwiftData model.

## Stash Borrowing

### `Stash/Stash/Services/ClipboardMonitor.swift`

Borrowing mode: **partial direct copy plus adaptation**.

Copy or closely adapt:

- `Timer.scheduledTimer(withTimeInterval: 0.3, repeats: true)` polling shape.
- `lastChangeCount` change detection.
- `markOwnChangeCount(_:)` concept for internal write guard.
- `topmostWindowOwner` and `sourceWindowOwnerPID(from:ownBundleID:ownPID:)`.
- `checkForChanges()` structure:
  - read current change count;
  - skip if paused;
  - read first pasteboard item;
  - determine source app;
  - detect content type;
  - extract plain text, URL, file paths, image data, rich text data;
  - call a change handler.
- `extractURL(from:)`.
- `extractFilePaths(from:)`.
- `extractImageData(from:item:)`.
- `detectWriterApp(filePaths:)`.
- `isLikelyImageFile(_:)`.

Adapt:

- Replace callback tuple with `ClipboardAssetCandidate`.
- Replace `excludedBundleIDs` privacy filtering with VoxFlow settings only if a user-facing exclusion setting remains. Phase one does not filter copied secrets by default.
- Keep internal write skip, because it prevents duplicate/noisy Assets.

Do not copy:

- Stash's 1Password/KeePassXC protected clipboard filtering from `PasteboardConstants` as a product default. The phase-one decision is to record user-copied secrets if the user copies them.

### `Stash/Stash/Model/ContentType.swift`

Borrowing mode: **direct copy with case rename**.

Copy the detection priority:

1. image
2. file URL
3. URL
4. rich text
5. plain text

Rename cases to VoxFlow's Asset language:

- `plainText` -> `text`
- `richText` -> `text` with `richTextData`
- `image` -> `image`
- `fileURL` -> `file`
- `url` -> `link`

Add `color` detection in VoxFlow after text extraction by matching CSS hex/RGB/HSL-style values. Do not add a separate Stash dependency for color detection.

### `Stash/Stash/Support/PasteboardConstants.swift`

Borrowing mode: **selective constant copy only**.

Copy technical marker raw values if not already copied from Maccy:

- `org.nspasteboard.ConcealedType`
- `org.nspasteboard.TransientType`
- `org.nspasteboard.AutoGeneratedType`

Do not copy password-manager markers as default exclusions.

### `Stash/Stash/Services/PasteService.swift`

Borrowing mode: **direct copy of switch structure, adapted to AssetActionService**.

Copy:

- text writes with `.string`;
- URL writes as plain string;
- image writes with `.png` data;
- rich text writes with `NSPasteboardItem`;
- file writes with `pasteboard.writeObjects(urls)`;
- `monitor.markOwnChangeCount(pasteboard.changeCount)` concept.

Adapt:

- Use VoxFlow's `ClipboardInternalWriteGuard` instead of Stash `ClipboardMonitor` directly.
- For text insertion, prefer existing `FastPasteTextInserter` after the pasteboard is prepared.

## Pasteboard Viewer Borrowing

### `Pasteboard Viewer/Pasteboard.swift`

Borrowing mode: **reference, optional small adaptation**.

Reference:

- `modernTypes` filtering of legacy pasteboard formats when a modern UTI equivalent exists.
- `com.apple.pasteboard.promised-suggested-file-name` hang avoidance.
- dynamic type ordering behavior.

Only copy the `typeExclusions` mapping if VoxFlow tests show duplicate/legacy type noise.

### `Pasteboard Viewer/ContentsScreen.swift`

Borrowing mode: **reference only**.

Use as a preview checklist:

- plain text preview;
- UTF-16 text decode;
- HTML text decode;
- image preview;
- RTF string preview;
- Quick Look fallback.

Do not copy this SwiftUI view into VoxFlow v1. The UI must follow the VoxFlow design mockup and existing app theme.

## VoxFlow Product Overrides

These decisions override borrowed defaults:

- User-copied secrets are recorded. Do not silently filter password-manager content by default.
- VoxFlow internal pasteboard writes are ignored to prevent duplicate/noisy Assets.
- OCR text is derived copyable text on a screenshot Asset. It is not a separate Asset.
- Pinning is out of scope for phase one.
- Ask AI attachment is out of scope for phase one.
